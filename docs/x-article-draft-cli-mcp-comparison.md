# X 記事ドラフト — note-research-cli

---

## 投稿本文

---

今日は、note.comで発信している人にとって「これ知らなかったやばい」ってなるツールの話をします。

それが「note-research-cli」です。

「なにそれ？」って人、多いと思います。

でもこれ、AIエージェントを使って情報発信を効率化したい人にとって、かなり刺さるやつなんです。

---

**何ができるかを先に言います。**

・競合ノートの検索・テーマ抽出
・人気記事のライク分布の分析
・自分の発信 vs 競合のギャップをレポート化
・分析結果からnoteの下書きを直接作成
・全部JSON/Markdownで出力できる

これを、ターミナルの1行コマンドでできます。

しかも、出力が全部JSON。

つまり「AIが読んで理解して、次のアクションを自分で考える」ために最適化された設計になっている。

---

**なぜCLI？という話**

そもそもnote.comを操作するMCPサーバーというものがすでに存在します。

Claude DesktopやCursorから自然言語でnoteを操作できる、多機能なやつです。

検索・分析・投稿・画像・コメント・統計。一通り全部入り。

「じゃあMCPでよくない？」

ちょっと待ってください。

MCPとCLIは、そもそも設計の思想が違います。

MCPは「エージェントが都度ツールを選んで実行する」もの。

CLIは「コマンドを直接叩く、または定型化して繰り返す」もの。

この違い、実はめちゃくちゃ大きいんです。

---

**本質的な差は「コンテキストの使い道」にある**

MCPは機能が多い分、エージェントが「どのツールを使うか」「どの引数を渡すか」を毎回考え続けます。

長い会話になるほど、その判断コストが積み上がっていく。

CLIはコマンドと出力だけ。

ツール選択に溶けていたコンテキストが、まるごと「分析結果の解釈」に使えます。

同じ競合調査を繰り返せば繰り返すほど、差が開く。

---

**実際の操作イメージ**

競合記事のテーマと人気度を分析したいとき：

```
$ note-research competitor analyze --query "生成AI" --size 40
```

自分の発信との差分を出したいとき：

```
$ note-research report gap \
  --mine-query "自分のテーマ" \
  --competitor-query "競合テーマ" \
  --format md
```

調査結果をJSONで蓄積して繰り返し再分析したいとき：

```
$ note-research report needs --query "LLM" --format json > result.json
```

これをcronに乗せれば、毎日自動で競合状況が手元に届きます。

---

**認証は初回だけ**

```
$ note-research auth login
```

セッションは `~/.note-research/session.json` に保存されます。

次からは何も考えずに実行できる。

ブラウザ自動ログイン・手動入力・環境変数の3モード対応。

CI/cronで安定運用したい場合は環境変数モードを使います。

---

**設計の思想が面白い**

「ローカルにインストールすれば、どのプロジェクトでもAIエージェントのツールとして使える」

MCPサーバーはクライアント設定や接続方式に依存します。

CLIは `npm install -g` して `auth login` するだけ。

シェルが動く環境ならどこでもエージェントのツールになる。

このコンセプト、個人的にすごく好きで。

「汎用的な基盤」を作るのではなく「目的に絞った道具」をローカルに置く発想。

これがエージェント連携の実質的なハードルを一番下げると思っています。

---

**使い分けの結論**

定型の競合調査（日次・バッチ・cron）→ CLI

対話で試行錯誤しながら投稿まで → MCP

併用するなら：

- 日次定型調査はCLIでcron
- 例外対応や試行はMCPで対話

この二層が一番バランスがいいと感じています。

